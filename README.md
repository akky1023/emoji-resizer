# Emoji Resizer (カスタム絵文字用 画像一括整形・リサイズツール)

長い方に合わせて正方形に整形、指定サイズにリサイズする。

## 要件

Windows 11以外では動作未確認

## ビルド

### npm (Recommended)
```bash
npm run build
```

### go コマンド
```bash
go build -o emoji-resizer.exe ./src
```
※ Go 1.18 以上。
※ この方法でビルドした場合、バージョン情報が `devel` になる

## 使い方

ビルドされた実行ファイル (`emoji-resizer.exe` または `emoji-resizer`) をbashで実行。

### 基本構文
対象の画像ファイルまたはディレクトリを指定。スペース区切り。
省略した場合は、カレントディレクトリ直下の画像を全て処理。

```bash
# 特定の画像をリサイズ
./emoji-resizer.exe image1.png image2.webp

# ディレクトリ直下の画像をすべてリサイズ
./emoji-resizer.exe path/to/images/
```

### オプション

* `-size`: リサイズ後の短辺（-rect指定時）または一辺、px単位 (デフォルト: 128)
* `-rect`: 短辺を-sizeに合わせてリサイズする。アスペクト比はそのまま（余白による正方形化を行わない）
* `-auto-rect`: アスペクト比（長辺/短辺）がしきい値を超える場合にのみ自動で `rect` モードとして扱い、それ以外は正方形（余白追加）として処理する。引数なしの場合は2.5をしきい値とする。数値を指定する場合（例: `-auto-rect=2.0`）は1より大きい値のみ指定可能。
* `-out`: 出力ディレクトリ指定 (デフォルト: output/)
* `-suffix`: 出力ファイル名に付与する接尾辞 (デフォルト: なし)
* `-name-prefix`: 絵文字名（および出力ファイル名）の前に付与する文字列 (デフォルト: なし)
* `-name-suffix`: 絵文字名（および出力ファイル名）の後ろに付与する文字列 (デフォルト: なし)
* `-r`: 再帰的に画像取得 
* `-no-resize`: リサイズしない (正方形または元のサイズにするだけモード)
* `-zip`: Misskey一括インポート用ZIPアーカイブ (`emojis.zip`) を生成する。日本語のファイル名に対しては対話式でひらがな表記の入力を求め、訓令式ローマ字での絵文字名と、ひらがな・カタカナ・ヘボン式ローマ字のエイリアスを自動設定する。実行時に対話式でカテゴリとライセンスの設定も可能。
* `-skip`: すでにリサイズ先の出力ファイルが存在する場合、その画像のリサイズ処理をスキップする
* `-config`: 設定ファイル（JSON）のパスを指定する。指定を省略した場合は、カレントディレクトリ直下に `config.json` があれば自動で読み込みます。
* `-check`: 変換対象をすべてチェックして変換後の絵文字名に被りがないか確認する (他オプションと併用可; 変換処理を**行わない**)
* `-version`: バージョン情報を表示して終了

### 設定ファイル (config.json)

オプションをファイルで指定することができます。コマンドライン引数で同じオプションが指定された場合は、コマンドライン引数の値が優先されます。

`config.json` の記述例:
```json
{
  "size": 128,
  "out": "output",
  "suffix": "_resized",
  "name_prefix": "pref_",
  "name_suffix": "_suf",
  "r": true,
  "no_resize": false,
  "rect": false,
  "zip": true,
  "auto_rect": 1.618,
  "skip": false,
  "category": "イラスト",
  "license": "CC-BY-4.0"
}
```

※ `category` および `license` を設定ファイルに記述しておくと、`-zip` 指定時の対話式プロンプト入力をスキップできます。

例
```bash
./emoji-resizer.exe -size 256 -suffix _resized sample.png
```
(なんかオプションは手前に付けないと怒られます。)
